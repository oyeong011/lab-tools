안정성 결론

현재 구성은 안정성을 우선한 실험 환경입니다. Ubuntu 공식 저장소 패키지 중심으로 설치했고, 커널 파라미터 변경, 오버클럭, 외부 드라이버 저장소, 위험한 전원 튜닝은 하지 않았습니다.

실험 실행은 기본적으로 낮은 CPU/I/O 우선순위, timeout, 온도 사전 점검, 실행 전후 센서 기록, 실행 중 모니터링 CSV를 사용하도록 구성했습니다.

지금까지 한 작업

tmux 개선

  • Ctrl-a를 tmux prefix로 설정
  • Ctrl-Left/Right로 session 이동
  • Ctrl-Up으로 새 session 생성
  • Ctrl-Down으로 session 목록 열기
  • 마우스 클릭/드래그/스크롤 지원

실험 디렉터리

  • ~/lab: 실험 작업공간
  • ~/lab/_hardware: 하드웨어 리포트
  • ~/lab/_benchmarks: 기준 벤치마크
  • ~/src: 소스 코드
  • ~/bin: 개인 명령어
  • ~/notes: 문서

안전 실행

  • 실행 전 온도 점검
  • 기본 timeout 10분
  • 낮은 CPU/I/O 우선순위
  • 실행 전후 센서 기록
  • 실행 중 monitor.csv 기록
  • suite 결과 summary.csv와 report.md 자동 생성
  • OpenCL GEMM 행렬곱 벤치 추가
  • NVIDIA GPU 장착 후 점검용 nvidia-check 추가

기본 사용법

새 실험 만들기

lab new my-test
lab go my-test

가속기 상태 확인

ai-accel-check

안전하게 OpenCL 벤치 실행

lab-safe-run cpu-ai-kernel-baseline -- bench-opencl

AI 연산 패턴에 가까운 GEMM 벤치 실행

lab-safe-run cpu-ai-kernel-baseline -- bench-opencl-gemm

CPU/OpenCL suite 3회 반복

bench-suite cpu-ai-kernel-baseline 3

warm-up 3회 + measured 10회 프로토콜

LAB_WARMUPS=3 LAB_REPEATS=10 bench-suite cpu-ai-kernel-baseline

YAML 선언형 suite 실행

bench-suite-config quick.yaml
bench-suite-config baseline.yaml

긴 실험을 5초 간격으로 모니터링

LAB_MONITOR_INTERVAL=5 monitor-run cpu-ai-kernel-baseline -- ./your-long-test

온도 임계값 변경

LAB_MAX_TEMP_C=80 lab-safe-run cpu-ai-kernel-baseline -- bench-opencl

나중에 NVIDIA GPU 장착 후 확인

nvidia-check
ai-accel-check

최근 검증값

항목값
OpenCL deviceIntel(R) HD Graphics 630
OpenCL vector add성공
OpenCL bandwidth 예시약 11 GB/s 전후
OpenCL GEMM 예시256x256 행렬곱 약 4 GFLOP/s 전후
STREAM 예시현재 샘플 설정에서 약 2.8 GB/s
OpenBLAS DGEMM 예시256x256, 4 threads 샘플에서 약 83 GFLOP/s
sysbench 예시2 threads, 2초 샘플에서 약 2700 events/s
CPU scalar_fma mean 예시약 0.54초
thermal guard 예시max_temp_c 약 50°C, threshold 85°C

남은 아쉬운 점

  • OpenCL GEMM은 추가됐지만 아직 naive 커널입니다. tiling/shared local memory 최적화 GEMM과 batch 변화 벤치가 더 필요합니다.
  • HD Graphics 630은 실험용 compute 장치로는 쓸 수 있지만, 최신 AI accelerator 수준의 성능은 기대하기 어렵습니다.
  • NVIDIA GPU가 들어오면 CUDA/cuDNN/ONNX Runtime GPU 경로를 별도로 추가해야 합니다.
  • SYCL/oneAPI 개발 환경은 아직 설치하지 않았습니다.
  • Docker/Podman 기반 격리 실험 환경은 아직 추가하지 않았습니다.
  • dotfiles 저장소화와 백업 자동화는 다음 단계입니다.

안전 정책

  • 기본적으로 lab-safe-run 또는 bench-suite로 실행합니다.
  • 온도가 높은 상태에서는 실행을 거부합니다.
  • 실험은 낮은 우선순위로 실행되어 데스크톱 사용성을 덜 방해합니다.
  • 각 run은 manifest.json에 커널, 패키지, OpenCL, 명령, 스크립트 checksum을 기록합니다.
  • 각 run은 result.json에 성공/실패, timeout, thermal abort 여부를 기록합니다.
  • 반복 suite는 stats.csv에 평균, 중앙값, 표준편차, CV, 품질 플래그를 기록합니다.
  • 커널 파라미터, 오버클럭, 외부 드라이버 저장소는 baseline이 충분히 쌓이기 전까지 피합니다.
  • 결과는 로그와 CSV로 남겨 비교 가능하게 관리합니다.
